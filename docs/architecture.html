System Architecture

Built on the MIAA Paradigm: Modularity, Isolation, Abstraction, Agnostic.

1. Modularity (Intelligence)

Models are defined as Capsules (YAML manifests). They are hot-swappable and decoupled from the runtime code.

  • Base Models: Ministral 3, GPT-OSS, DeepSeek.
  • Adapters: Dynamic LoRA injection (supported).
2. Isolation (Execution)

Inference runs in strictly isolated Docker containers using hardware-specific profiles.

  • CUDA Profile: NVIDIA runtime.
  • ROCm Profile: AMD runtime.
  • Benefit: A crash in the GPU layer does not take down the API gateway.
3. Abstraction (Middleware)

A "Universal Translator" Middleware (FastAPI) normalizes all traffic.

  • Unified Schema: Enforces standard inputs/outputs.
  • Embedding Adaptation: (Future) Maps vector spaces.
4. Agnostic (Interface)

The system connects to the world via standard protocols, not custom code.

  • MCP Host: Connects to Model Context Protocol servers.
  • Observability: OpenLLMetry/OpenTelemetry hooks.
